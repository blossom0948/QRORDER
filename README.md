# QRORDER

스마트 QR 테이블오더 MVP 웹사이트입니다.

## 화면 구조

- `index.html`: 손님용 주문 화면
- `admin.html`: 사장님용 주문/메뉴 관리 화면

손님 화면에는 관리자 기능이나 사이트 소개 영역을 노출하지 않고, 메뉴, 장바구니, 주문 확인, 내 주문 진행상황만 제공합니다.

## 기능

- 손님: 메뉴 조회, 검색, 장바구니, 함께 담기 좋은 메뉴 추천, 주문 접수
- 손님: 인기/추천 메뉴 빠른 담기
- 손님: 메뉴 상세 하단 시트와 기본 옵션 선택
- 손님: 한국어, 영어, 일본어, 중국어 보기 모드
- 손님: 고정 문구, 메뉴명, 설명, 옵션, 장바구니, 주문 완료 화면 다국어 전환
- 손님: 직접 추가한 메뉴의 내장 음식 사전 기반 즉시 번역
- 손님: 큰 글씨 접근성 모드
- 손님: QR 토큰 확인 상태와 실시간 동기화 상태 표시
- 손님: 모바일 하단 주문 확인 바
- 손님: 주문 완료 후 5초 카운트다운 뒤 주문 화면 자동 복귀
- 손님: 현재 주문한 메뉴의 접수, 조리중, 완료 진행상황 확인
- 손님: 진행 중인 주문 화면에서 물, 앞접시, 냅킨 직원 호출
- 손님: 후불/선불 준비 결제 모드와 주문/환불 안내 시트
- 사장님: 주문 대기, 조리중, 완료 상태 관리
- 사장님: 주문 보드를 화면 최상단에 배치
- 사장님: 새 주문/직원 호출 접수 시 딩동 알림
- 사장님: 전체, 음식 주문, 직원 호출 필터
- 사장님: 진행 테이블, 직원 호출, 품절 메뉴, 최장 대기 시간 운영 요약
- 사장님: 실시간, QR 보안, 메뉴 사진, 결제 연동 준비 상태 확인
- 사장님: 음식 주문과 직원 호출 요청 구분 확인
- 사장님: 메뉴 이미지 URL/파일 업로드, 이름, 설명, 분류, 가격, 배지, 옵션 그룹, 품절 여부 설정
- 사장님: 메뉴 노출 순서 변경
- 사장님: 메뉴 추가 및 삭제
- Firebase: 사장님 로그인, 매장 생성, 기본 메뉴 생성, 테이블 QR 생성, 실시간 주문 동기화

URL에 `store`와 `token`이 있으면 Firebase Firestore 실매장 모드로 동작합니다. 없으면 기존 localStorage 데모 모드로 동작합니다.

Firebase 무료 기준 운영 순서는 [FREE_SETUP_GUIDE.md](FREE_SETUP_GUIDE.md)에 정리했습니다. Firestore 보안 규칙은 [firestore.rules](firestore.rules)를 Firebase Console에 붙여 넣으면 됩니다.
